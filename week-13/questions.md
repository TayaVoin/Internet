# Вопросы для самопроверки

## 1. Зачем нужен Helm? Какую проблему он решает?
**Helm** — это пакетный менеджер для Kubernetes, как apt для Ubuntu или pip для Python.
**Проблемы, которые решает Helm:**
| Проблема | Решение Helm |
|----------|--------------|
| **Повторяемость** | Один и тот же код для dev/stage/prod |
| **Копипаста YAML** | Шаблоны вместо копирования |
| **Управление версиями** | Версионирование чартов |
| **Сложность обновлений** | `helm upgrade` и `helm rollback` |
| **Разные окружения** | Values-файлы для dev/stage/prod |
## 2. Как работает приоритет значений (Values)? Кто победит: `values.yaml` или флаг `--set`?
Порядок приоритета (от низшего к высшему):
1. values.yaml (дефолтные)
2. values-<env>.yaml (переопределения окружений)
3. --set флаги (наивысший приоритет)
Пример:
```bash
# values.yaml: replicaCount=1
# values-prod.yaml: replicaCount=3
# --set replicaCount=5

helm template ./chart -f values-prod.yaml --set replicaCount=5
# Результат: replicaCount=5 (победил --set)
```
## 3. Что такое функция `include` и `_helpers.tpl`? Зачем они нужны?
`_helpers.tpl` — файл для хранения общих шаблонов и функций.
`include` — функция для вставки этих шаблонов.
**Зачем:**
- DRY (Don't Repeat Yourself) — метки не дублируются
- Единообразие — одинаковые метки во всех ресурсах
- Удобство — изменения в одном месте
## 4. Почему в шаблонах Helm так важно следить за отступами и зачем нужна функция `indent` или `nindent`?
YAML чувствителен к пробелам. Шаблонизатор просто генерирует текст, а неправильные отступы сломают YAML.
`nindent` X — добавляет новую строку и X пробелов
`indent` X — добавляет X пробелов к каждой строке
## 5. Как откатить неудачный релиз с помощью Helm?
**Команды:**
```bash
# Посмотреть историю релизов
helm history my-release

# Откатиться к предыдущей версии
helm rollback my-release 1  # откат к версии 1

# Посмотреть статус после отката
helm status my-release
```
## 6. Можно ли использовать Helm только для генерации YAML, без установки в кластер?
- ДА! Helm умеет рендерить шаблоны без установки.
- **Команда helm template:**
```bash
# Сгенерировать YAML для dev окружения
helm template my-release ./chart -f values-dev.yaml > dev-manifests.yaml

# Сгенерировать для prod
helm template my-release ./chart -f values-prod.yaml > prod-manifests.yaml
```
- **Где это полезно:**
    - CI/CD пайплайны — можно сгенерировать YAML и сохранить как артефакт
    - Проверка шаблонов — увидеть результат до применения
    - GitOps — коммитить готовые YAML в репозиторий
    - Интеграция с другими инструментами — передать YAML в другой инструмент
