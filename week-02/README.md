# Полный CRUD и валидация

## Задача
На прошлой неделе мы научились создавать и получать объекты. Теперь время довести дело до конца: реализуем полноценный CRUD (Create, Read, Update, Delete). Кроме того, научим наш сервис проверять данные (валидация) и корректно сообщать об ошибках.

## Ваш вариант
Используйте тот же вариант, что и на первой неделе:
`variants/<GROUP>/<STUDENT_ID>/week-02.json` (сущность и поля совпадают с неделей 01).

Не забудьте настроить переменные, если открыли новый терминал:
```bash
export GROUP=431
export STUDENT_ID=s01
```

## Что нужно сделать
1. **Добавить методы**:
   - `PUT /<resource>/{id}` — полное обновление объекта.
   - `DELETE /<resource>/{id}` — удаление объекта.
2. **Настроить валидацию**:
   - Используйте Pydantic модели.
   - Если клиент присылает неправильный тип данных (строку вместо числа) — возвращайте понятную ошибку (код 422).
   - Если объект не найден по ID — возвращайте 404.
3. **Написать тесты**:
   - В папке `tests/` добавьте свой файл с тестами (например, `test_crud.py`).
   - Напишите тест на успешное обновление.
   - Напишите тест на ошибку 404 при удалении несуществующего объекта.

## Что сдавать
1. Архив с кодом.
2. Ответы на вопросы.
3. Скриншот или лог прохождения тестов.

## Как проверить
```bash
make test WEEK=02
```
Тесты этой недели строже: они проверяют коды ошибок и форматы ответов.
