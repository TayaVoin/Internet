## Выводы
1. **gRPC быстрее REST** примерно в 2.4 раза для Unary вызовов благодаря:
   - Бинарному формату Protobuf (меньше данных для парсинга)
   - HTTP/2 мультиплексированию (одно соединение на все запросы)
   - Меньшим накладным расходам на сериализацию
2. **Конкурентность** значительно увеличивает RPS для REST (с 176 до 1215), но gRPC все равно выигрывает.
3. **Server Streaming** работает как ожидалось - сервер отправляет поток сообщений с задержкой, клиент получает их по мере готовности. Это невозможно в REST без polling или WebSocket.
4. **Latency**:
   - gRPC: стабильно низкая (2-7 ms)
   - REST: сильно варьируется (5-23 ms)
5. **RPS (Requests Per Second)**:
   - gRPC: ~4200
   - REST конкурентно: ~1200
   - REST синхронно: ~175
