# Вопросы для самопроверки

## 1. Что такое слой в Docker и почему порядок команд в Dockerfile важен для кэширования?
**Слой (Layer)** — это результат выполнения одной инструкции в Dockerfile (`FROM`, `RUN`, `COPY`, `ADD`). Каждый слой сохраняется в кэше и может быть переиспользован при следующих сборках.
**Почему порядок важен:**
- Docker кэширует каждый слой
- Если слой изменился, все последующие слои пересобираются
- Часто изменяемые инструкции должны идти в конце
## 2. В чем суть Multi-stage build? Какие проблемы он решает?
**Multi-stage build** — это использование нескольких FROM в одном Dockerfile. Каждый этап может использовать разные базовые образы.
**Проблемы, которые решает:**
- Размер образа: В финальный образ попадает только то, что нужно для работы
- Безопасность: Инструменты сборки (gcc, компиляторы) не попадают в продакшен
- Читаемость: Всё в одном файле, не нужно несколько Dockerfile
## 3. Зачем нужен файл `.dockerignore`?
**.dockerignore** работает как .gitignore — исключает файлы из контекста сборки.
**Зачем нужен:**
- Скорость: Меньше данных отправляется Docker daemon
- Безопасность: Секреты (.env, ключи) не попадают в образ
- Размер: Исключаются временные файлы, кэш, виртуальные окружения
## 4. Чем отличаются инструкции `CMD` и `ENTRYPOINT`?
| Характеристика | `CMD` | `ENTRYPOINT` |
|----------------|-------|--------------|
| **Назначение** | Задает команду и аргументы по умолчанию | Задает исполняемый файл, который всегда будет запущен |
| **Переопределение** | Легко переопределяется полностью при запуске контейнера | Трудно переопределить, можно только дополнить аргументами |
| **Формат** | `CMD ["executable", "param1", "param2"]` (exec-форма)<br>`CMD command param1 param2` (shell-форма) | `ENTRYPOINT ["executable", "param1"]` (exec-форма)<br>`ENTRYPOINT command param1` (shell-форма) |
| **Запуск с переопределением** | `docker run ... /bin/bash` — заменит `CMD` на `/bin/bash` | `docker run ... --host 0.0.0.0 --port 8000` — аргументы добавятся к `ENTRYPOINT` |
## 5. Почему рекомендуется использовать `COPY` вместо `ADD`?
- **Предсказуемость:** COPY всегда делает одно и то же
- **Безопасность:** Не скачивает файлы из интернета
- **Размер:** Не распаковывает неожиданные архивы
## 6. Почему приложение в контейнере должно слушать `0.0.0.0`, а не `127.0.0.1`?
- **127.0.0.1 (localhost)** слушает только внутри контейнера — снаружи не достучаться.
- **0.0.0.0** слушает на всех сетевых интерфейсах:
    - Внутри контейнера: localhost
    - Между контейнерами: bridge network
    - С хоста: проброшенный порт
