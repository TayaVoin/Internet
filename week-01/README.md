# Первый микросервис и REST API

## Задача
На этой неделе мы создадим свой первый микросервис на FastAPI. Цель — разобраться, как работает REST API, научиться принимать запросы и отдавать корректные ответы. Никаких сложных баз данных пока не нужно, всё храним в памяти (в обычной переменной).

## Ваш вариант
У каждого — своё задание. Найдите свой файл в папке `variants/`:
`variants/<GROUP>/<STUDENT_ID>/week-01.json`

В файле указано:
- Какой ресурс вы создаёте (например, `book` или `user`).
- Какое дополнительное поле должно быть у этого ресурса.

Перед запуском не забудьте настроить переменные:
```bash
export GROUP=431  # Ваша группа
export STUDENT_ID=s01 # Ваш ID
```

## Что нужно сделать
Напишите сервис на FastAPI, который реализует три метода для вашего ресурса:
1. `GET /<resource>` — вернуть список всех объектов.
2. `POST /<resource>` — создать новый объект.
3. `GET /<resource>/{id}` — получить один объект по его ID.

Важно:
- Используйте in-memory хранилище (список или словарь).
- При перезапуске сервера данные могут пропадать — это нормально.

## Что сдавать
1. Архив с вашим кодом.
2. Файл с ответами на вопросы из `questions.md` (кратко, своими словами).
3. Инструкцию, как это всё запустить (если отличается от стандартной).

## Как проверить себя
Просто запустите тесты:
```bash
make test WEEK=01
```
Если тесты проходят — вы молодцы!
